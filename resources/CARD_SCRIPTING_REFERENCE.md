## File Structure

Each card is a plain text file with key-value lines. A minimal card:

```
Name:Grizzly Bears
ManaCost:1 G
Types:Creature Bear
PT:2/2
Oracle:
```

Lines are parsed as `Key:Value` pairs. The order of lines matters only for readability — put abilities (A/T/S/R), then SVars, then AI SVars, then Oracle last.

---

## Top-Level Properties

| Property | Description | Example |
|----------|-------------|---------|
| `Name` | Card name (required). Trademark symbols cannot be included. | `Name:Lightning Bolt` |
| `ManaCost` | Mana cost in space-separated shards. Use `no cost` for cards that can't be cast. | `ManaCost:2 W W` |
| `Types` | All supertypes, card types, and subtypes separated by spaces. | `Types:Legendary Creature Human Wizard` |
| `PT` | Power/toughness for creatures. | `PT:3/4` |
| `Loyalty` | Starting loyalty for planeswalkers. | `Loyalty:3` |
| `Colors` | Override color (for cards whose color differs from mana cost, e.g., back faces). | `Colors:red,green` |
| `Text` | Additional card text not generated by abilities (e.g., "CARDNAME can be your commander."). | `Text:X can't be 0.` |
| `Oracle` | Current Oracle rules text. Should always be the last line. | `Oracle:Deal 3 damage to any target.` |
| `AlternateMode` | Multi-face layout type (see [Multi-Face Cards](#multi-face-cards)). | `AlternateMode:DoubleFaced` |

### Mana Cost Shards

| Shard | Meaning | Example |
|-------|---------|---------|
| `W` `U` `B` `R` `G` | Colored mana | `ManaCost:1 W W` |
| `C` | Colorless mana | `ManaCost:2 C` |
| `S` | Snow mana | `ManaCost:S S` |
| `1` `2` ... `N` | Generic mana | `ManaCost:3` |
| `X` | Variable | `ManaCost:X R` |
| `WU` `WB` `UB` etc. | Hybrid | `ManaCost:2 WU WU` |
| `WP` `UP` `BP` `RP` `GP` | Phyrexian | `ManaCost:UP UP` |
| `W2` `U2` etc. | Two-or-colored hybrid | `ManaCost:W2 U2` |
| `no cost` | Card cannot be cast | `ManaCost:no cost` |

## Multi-Face Cards

Cards with multiple faces separate faces with a blank line followed by `ALTERNATE`. The front face specifies the layout with `AlternateMode`:

| AlternateMode | Used For |
|---------------|----------|
| `DoubleFaced` | Transform cards (DFCs) |
| `Split` | Split cards (Fire // Ice) |
| `Adventure` | Adventure cards |
| `Modal` | Modal double-faced cards (MDFCs) |
| `Flip` | Kamigawa flip cards |

**Transform card example (Delver of Secrets):**
```
Name:Delver of Secrets
ManaCost:U
Types:Creature Human Wizard
PT:1/1
T:Mode$ Phase | Phase$ Upkeep | ValidPlayer$ You | ...
AlternateMode:DoubleFaced
Oracle:...

ALTERNATE

Name:Insectile Aberration
ManaCost:no cost
Colors:blue
Types:Creature Human Insect
PT:3/2
K:Flying
Oracle:Flying
```

**Split card example (Fire // Ice):**
```
Name:Fire
ManaCost:1 R
Types:Instant
A:SP$ DealDamage | ...
AlternateMode:Split
Oracle:...

ALTERNATE

Name:Ice
ManaCost:1 U
Types:Instant
A:SP$ Tap | ...
Oracle:...
```

**Adventure card example (Bonecrusher Giant // Stomp):**
```
Name:Bonecrusher Giant
ManaCost:2 R
Types:Creature Giant
PT:4/3
T:...
AlternateMode:Adventure
Oracle:...

ALTERNATE

Name:Stomp
ManaCost:1 R
Types:Instant Adventure
A:SP$ Effect | ...
Oracle:...
```

## Keywords

Keywords are declared with the `K:` prefix, one per line.

### Keywords Without Parameters

Simple keywords that match their printed name:

```
K:Flying
K:First Strike
K:Haste
K:Trample
K:Vigilance
K:Deathtouch
K:Lifelink
K:Reach
K:Flash
K:Indestructible
K:Menace
K:Cascade
K:Convoke
K:Devoid
K:Split second
K:Undying
K:Wither
```

### Special Keyword: ETB Counters

```
K:etbCounter:P1P1:X
```
Places counters when the card enters the battlefield. Used by cards like Walking Ballista.

### Plaintext Keywords

Some pseudo-keywords are hardcoded text rather than true keyword abilities. `CARDNAME` is replaced by the card's name:

```
K:CARDNAME can't attack or block alone.
K:CARDNAME must be blocked if able.
K:You may choose not to untap CARDNAME during your untap step.
```

## Abilities

Abilities are declared with the `A:` prefix and follow AbilityFactory syntax:

```
A:<Type>$ <ApiType> | Param1$ Value1 | Param2$ Value2 | SpellDescription$ text
```

Parameters are pipe-delimited `Key$ Value` pairs.

---

## Ability Types and Prefixes

| Prefix | Name | Usage |
|--------|------|-------|
| `AB` | Activated Ability | Abilities with costs that can be activated by the controller |
| `SP` | Spell | Cast from hand (instants, sorceries, and spell-like effects) |
| `DB` | Drawback/SubAbility | Chained sub-effects; never appears directly on a card as a root ability |
| `ST` | Static | Resolves immediately without using the stack (rare) |

### ApiType (Effect Types)

The word after `$` is the `ApiType` — the specific effect. There are 200+ types. Common ones:

**Damage:** `DealDamage`, `DamageAll`, `DamageEach`
**Card Draw:** `Draw`, `Mill`, `Dig`, `DigUntil`, `PeekAndReveal`
**Life:** `GainLife`, `LoseLife`, `SetLife`
**Zones:** `ChangeZone`, `ChangeZoneAll`
**Counters:** `PutCounter`, `RemoveCounter`, `MoveCounter`, `Proliferate`
**Creatures:** `Token`, `Pump`, `PumpAll`, `Animate`, `Fight`
**Removal:** `Destroy`, `DestroyAll`, `Sacrifice`, `SacrificeAll`
**Stack:** `Counter`
**Mana:** `Mana`
**Control:** `GainControl`, `ExchangeControl`
**State:** `Tap`, `TapAll`, `Untap`, `UntapAll`, `SetState`, `Phases`
**Cards:** `Discard`, `Reveal`, `RevealHand`, `Shuffle`
**Modal:** `Charm`
**Continuous:** `Effect`
**Copy:** `Clone`, `CopyPermanent`, `CopySpellAbility`
**Protection:** `Protection`, `PreventDamage`, `Fog`, `Regenerate`
**Choice:** `ChooseCard`, `ChooseColor`, `ChooseNumber`, `ChoosePlayer`, `ChooseType`, `NameCard`
**Logic:** `Branch`, `Repeat`, `RepeatEach`, `Cleanup`
**Game:** `GameWin`, `GameLoss`, `GameDraw`, `RestartGame`
**Turn:** `AddTurn`, `SkipTurn`, `AddPhase`, `EndTurn`
**Random:** `FlipACoin`, `RollDice`
**Play:** `Play`
**Special:** `Amass`, `Investigate`, `Explore`, `Goad`, `Vote`, `StoreSVar`

The full list of ApiTypes maps 1:1 with classes in `forge-game/src/main/java/forge/game/ability/effects/`.

## Cost Syntax

Costs appear in the `Cost$` parameter, with parts separated by spaces.

### Basic Costs

| Token | Meaning | Example |
|-------|---------|---------|
| `T` | Tap this permanent | `Cost$ T` |
| `Q` | Untap this permanent | `Cost$ Q` |
| Mana shards | Pay mana | `Cost$ 2 R` |
| `0` | Zero cost (free to activate) | `Cost$ 0` |

### Parameterized Costs

Format: `CostType<Amount/Type/Description>`

| Cost | Syntax | Example |
|------|--------|---------|
| Pay life | `PayLife<N>` | `Cost$ PayLife<3>` |
| Pay energy | `PayEnergy<N>` | `Cost$ PayEnergy<2>` |
| Sacrifice | `Sac<N/Valid/Desc>` | `Cost$ Sac<1/Creature>` |
| Discard | `Discard<N/Type/Desc>` | `Cost$ Discard<1/Card>` |
| Exile (battlefield) | `Exile<N/Valid/Desc>` | `Cost$ Exile<1/CARDNAME>` |
| Exile (graveyard) | `ExileFromGrave<N/Valid>` | `Cost$ ExileFromGrave<2/Card>` |
| Exile (hand) | `ExileFromHand<N/Valid>` | `Cost$ ExileFromHand<1/Card>` |
| Exile (top of library) | `ExileFromTop<N/Valid>` | `Cost$ ExileFromTop<3/Card>` |
| Return to hand | `Return<N/Valid/Desc>` | `Cost$ Return<1/Creature>` |
| Remove counter | `SubCounter<N/Type>` | `Cost$ SubCounter<1/P1P1>` |
| Add counter | `AddCounter<N/Type>` | `Cost$ AddCounter<2/LOYALTY>` |
| Tap other permanents | `tapXType<N/Valid/Desc>` | `Cost$ tapXType<3/Creature>` |
| Mill | `Mill<N>` | `Cost$ Mill<3>` |
| Reveal | `Reveal<N/Valid>` | `Cost$ Reveal<1/Dragon>` |
| Restricted mana | `Mana<N\Valid>` | `Cost$ Mana<2\Creature>` |

Use `CARDNAME` as the type for self-referencing costs: `Sac<1/CARDNAME>`, `Exile<1/CARDNAME>`.

Counter types are ALL CAPS: `P1P1` (+1/+1), `M1M1` (-1/-1), `LOYALTY`, `CHARGE`, `LORE`, etc.

### Planeswalker Costs

Planeswalker loyalty costs use counter costs:
- **+N loyalty:** `Cost$ AddCounter<N/LOYALTY>` with `Planeswalker$ True`
- **-N loyalty:** `Cost$ SubCounter<N/LOYALTY>` with `Planeswalker$ True`
- **0 loyalty:** `Cost$ AddCounter<0/LOYALTY>` with `Planeswalker$ True`

### Combined Costs

Multiple cost parts are space-separated:

```
Cost$ 1 G T                          -- Pay {1}{G} and tap
Cost$ T Sac<1/CARDNAME>              -- Tap and sacrifice self
Cost$ SubCounter<1/P1P1>             -- Remove a +1/+1 counter
Cost$ 2 R Discard<1/Card>            -- Pay {2}{R} and discard a card
```

### UnlessCost

`UnlessCost$` allows a player to pay to prevent an effect:

```
UnlessCost$ 2                        -- Unless that player pays {2}
UnlessPayer$ Targeted                -- Who must pay (default: TargetedController)
UnlessSwitched$ True                 -- Inverts: pay to resolve instead of prevent
```

## Targeting and Affected Objects

### Targeting (ValidTgts$)

For abilities that target, use `ValidTgts$`:

```
A:SP$ DealDamage | ValidTgts$ Creature | NumDmg$ 3 | SpellDescription$ ...
A:SP$ Destroy | ValidTgts$ Artifact,Enchantment | SpellDescription$ ...
```

Additional targeting parameters:

| Parameter | Purpose | Example |
|-----------|---------|---------|
| `ValidTgts$` | What can be targeted (Valid expression) | `ValidTgts$ Creature.YouCtrl` |
| `TgtPrompt$` | Custom prompt text (usually auto-generated) | `TgtPrompt$ Select target creature` |
| `TgtZone$` | Zone to target from (default: Battlefield) | `TgtZone$ Graveyard` |
| `TargetMin$` | Minimum number of targets | `TargetMin$ 1` |
| `TargetMax$` | Maximum number of targets | `TargetMax$ 3` |
| `TargetingPlayer$` | Player who chooses targets | `TargetingPlayer$ Opponent` |
| `DividedAsYouChoose$` | Total to divide among targets | `DividedAsYouChoose$ 3` |

`Any` is a special target that means "any target" (creature, player, or planeswalker).

## Valid Expressions

Valid expressions filter cards and players throughout the scripting language. They appear in `ValidTgts$`, `ValidCard$`, `Affected$`, `SacValid$`, `ChangeType$`, and many other parameters.

### Format

```
<Type>[.restriction1[+restriction2]][,<AltType>[.restriction]]
```

- Comma (`,`) means OR between types
- Dot (`.`) separates type from restrictions
- Plus (`+`) means AND between restrictions
- Prefix `!` negates a restriction

### Card Types

`Card`, `Permanent`, `Creature`, `Artifact`, `Enchantment`, `Land`, `Instant`, `Sorcery`, `Planeswalker`, `Battle`, and any subtype (e.g., `Goblin`, `Aura`, `Equipment`, `Forest`).

### Common Restrictions

| Restriction | Meaning |
|-------------|---------|
| `YouCtrl` | Controlled by ability's controller |
| `OppCtrl` | Controlled by an opponent |
| `YouOwn` | Owned by ability's controller |
| `Other` | Not the source card |
| `Self` | Is the source card |
| `tapped` / `untapped` | Tap state |
| `token` / `!token` | Is/isn't a token |
| `attacking` / `blocking` | Combat state |
| `faceDown` | Is face down |
| `enchanted` / `equipped` | Has aura/equipment |
| `Black`, `Blue`, `White`, `Red`, `Green` | Has color |
| `Colorless` / `Multicolor` | Color properties |
| `withFlying`, `withHaste`, etc. | Has keyword |
| `named{CardName}` | Specific card name |
| `ChosenType` / `ChosenColor` | Matches a prior choice |
| `IsRemembered` | Is in the remembered list |
| `power{cmp}{N}` | Power comparison (e.g., `powerGE4`) |
| `toughness{cmp}{N}` | Toughness comparison |
| `cmc{cmp}{N}` | CMC comparison |
| `counters{cmp}{N}{Type}` | Counter comparison (e.g., `countersGE4LEVEL`) |

### Comparators

Used in numeric restrictions: `LT` (less than), `LE` (<=), `EQ` (equal), `NE` (not equal), `GE` (>=), `GT` (greater than).

### Examples

```
Creature                           -- Any creature
Creature.Other                     -- Another creature (not source)
Artifact,Enchantment               -- Artifact OR enchantment
Creature.!Black+!Artifact          -- Non-black non-artifact creature
Creature.withFlying                -- Creature with flying
Creature.YouCtrl+powerGE4          -- Creature you control with power >= 4
Card.IsRemembered                  -- A remembered card
Permanent.OppCtrl                  -- Permanent an opponent controls
Land.Basic                         -- Basic land
```

For the full set of properties, see `CardProperty.java` and `PlayerProperty.java`.

## Defined$ References

`Defined$` specifies non-targeted objects affected by an ability:

### Players

| Value | Meaning |
|-------|---------|
| `You` | Controller of the ability (default for many effects) |
| `Opponent` / `OpponentNotOwner` | Opponent(s) |
| `Player` | All players |
| `TargetedController` / `TargetedOwner` | Controller/owner of targeted object |
| `AttackingPlayer` / `DefendingPlayer` | Combat participants |
| `TriggeredPlayer` / `TriggeredActivator` | Player from trigger context |
| `EnchantedController` | Controller of enchanted permanent |

### Cards

| Value | Meaning |
|-------|---------|
| `Self` | The source card (default) |
| `Equipped` / `Enchanted` | Card this is attached to |
| `Targeted` / `ParentTarget` | Card targeted by this/parent ability |
| `Remembered` | Cards in the remembered list |
| `Imprinted` | Cards in the imprinted list |
| `ChosenCard` | Card chosen via ChooseCard |
| `TopOfLibrary` / `BottomOfLibrary` | Library position |
| `ExiledWith` | Cards exiled with this card |
| `TriggeredCard` / `TriggeredAttacker` etc. | Objects from trigger context |
| `ReplacedCard` | Object from replacement context |

### Valid Defined

Use `Valid` to define by restriction instead of identity:

```
Defined$ ValidBattlefield Creature.YouCtrl
Defined$ ValidGraveyard Creature.YouOwn+ThisTurnEnteredFrom_Battlefield
Defined$ ValidAll Card
```

## Triggered Abilities

Triggers are declared with the `T:` prefix:

```
T:Mode$ <TriggerType> | <Parameters> | Execute$ <SVar> | TriggerDescription$ <text>
```

### Key Parameters

| Parameter | Purpose |
|-----------|---------|
| `Mode$` | The game event that fires this trigger |
| `Execute$` | SVar name containing the ability to run |
| `TriggerZones$` | Zone the card must be in (default: all zones) |
| `TriggerDescription$` | Display text for the trigger |
| `Static$ True` | Resolves immediately, not via stack (for "as ~ enters" effects) |
| `Secondary$ True` | Hides description from card text |
| `OptionalDecider` | Player who chooses whether to trigger |
| `NoResolvingCheck$ True` | Don't recheck condition on resolution |

### Common Trigger Modes

| Mode | Fires When | Key Parameters | TriggeredObjects |
|------|-----------|----------------|------------------|
| `ChangesZone` | Card moves between zones | `ValidCard$`, `Origin$`, `Destination$` | `Card`, `CardLKI` |
| `Phase` | A phase begins | `Phase$`, `ValidPlayer$` | `Player` |
| `SpellCast` | A spell is cast | `ValidCard$`, `ValidActivatingPlayer$` | `Card`, `SpellAbility`, `Player` |
| `Attacks` | A creature attacks | `ValidCard$`, `Alone$` | `Attacker` |
| `Blocks` | A creature blocks | `ValidCard$`, `ValidBlocked$` | `Attacker`, `Blocker` |
| `DamageDone` | Damage is dealt | `ValidSource$`, `ValidTarget$`, `CombatDamage$` | `Source`, `Target`, `DamageAmount` |
| `BecomesTarget` | Something is targeted | `ValidSource$`, `ValidTarget$` | `Source`, `Target` |
| `CounterAdded` | Counter placed on card | `ValidCard$`, `CounterType$` | `Card` |
| `LifeGained` / `LifeLost` | Life total changes | `ValidPlayer$` | `Player`, `LifeAmount` |
| `Drawn` | Card is drawn | `ValidCard$` | `Card`, `Player` |
| `Discarded` | Card is discarded | `ValidPlayer$`, `ValidCard$` | `Card` |
| `Sacrificed` | Permanent is sacrificed | `ValidPlayer$`, `ValidCard$` | `Card` |
| `Destroyed` | Permanent is destroyed | `ValidCard$` | `Card` |
| `TapsForMana` | Land tapped for mana | `ValidCard$` | `Card`, `Player`, `Produced` |
| `Taps` / `Untaps` | Permanent taps/untaps | `ValidCard$` | `Card` |
| `LandPlayed` | Land played | `ValidCard$` | `Card` |
| `Transformed` | Card transforms | `ValidCard$` | `Transformer` |
| `Always` | State trigger (CR 603.8) | | |

### Accessing Triggered Variables

Trigger context values are accessed via `Triggered` prefix:

```
Defined$ TriggeredCard              -- The card from the trigger event
Defined$ TriggeredAttacker          -- The attacking creature
Defined$ TriggeredSourceController  -- Controller of the trigger source
SVar:X:TriggeredCard$CardPower      -- Power of the triggered card
SVar:X:TriggerCount$TriggeredLifeAmount  -- Life amount from trigger
```

Append `Controller` or `Owner` to card variables to get the corresponding player.

## Static Abilities

Static abilities are declared with the `S:` prefix. There are two major categories:

### Continuous Effects (Layer System)

```
S:Mode$ Continuous | Affected$ <Valid> | <LayerParams> | Description$ <text>
```

These implement Magic's 7-layer system for continuous effects:

| Layer | Effect | Parameters |
|-------|--------|------------|
| 2 | Control change | `GainControl$` |
| 3 | Text change | `ChangeColorWord$`, `ChangeTypeWord$` |
| 4a | Add/remove types | `AddType$`, `RemoveType$` |
| 4b | Add/remove abilities | `AddKeyword$`, `AddAbility$`, `AddTrigger$` |
| 5 | Set P/T | `SetPower$`, `SetToughness$` |
| 6 | Modify P/T | `AddPower$`, `AddToughness$` |
| 7 | Switch P/T | `SwitchPT$ True` |

```
S:Mode$ Continuous | Affected$ Creature.YouCtrl | AddPower$ 1 | AddToughness$ 1 | Description$ Creatures you control get +1/+1.
S:Mode$ Continuous | Affected$ Creature.YouCtrl | AddKeyword$ Flying & Indestructible | Description$ ...
S:Mode$ Continuous | Affected$ You | SetMaxHandSize$ Unlimited | Description$ You have no maximum hand size.
```

Multiple keywords are `&`-delimited: `AddKeyword$ Flying & Haste & Trample`

Key optional parameters:
- `AffectedZone$` — zone the affected cards must be in (default: Battlefield)
- `EffectZone$` — zone the source must be in for the static to work (default: Battlefield)
- `CharacteristicDefining$ True` — marks as a characteristic-defining ability (layer 4)

### Game Rule Modifications

These use specific Mode values instead of `Continuous`:

| Mode | Purpose | Example |
|------|---------|---------|
| `CantBeCast` | Prevents casting | `S:Mode$ CantBeCast \| ValidCard$ ...` |
| `CantBeActivated` | Prevents activation | |
| `CantPlayLand` | Prevents land plays | |
| `CantAttack` / `CantBlock` | Combat restrictions | |
| `MustAttack` / `MustBlock` | Forced combat | |
| `CantGainLife` | Prevents life gain | |
| `CantPreventDamage` | Prevents damage prevention | |
| `AlternativeCost` | Alternative casting costs | |
| `RaiseCost` / `ReduceCost` / `SetCost` | Cost modification | |
| `CantAttackUnless` / `CantBlockUnless` | Conditional combat costs | |
| `OptionalCost` | Optional additional costs | |
| `DisableTriggers` | Prevents triggers from firing | |
| `Panharmonicon` | Doubles triggered abilities | |

## Replacement Effects

Replacement effects are declared with the `R:` prefix:

```
R:Event$ <ReplacementType> | <Parameters> | ReplaceWith$ <SVar> | Description$ <text>
```

### Key Parameters

| Parameter | Purpose |
|-----------|---------|
| `Event$` | The game event being replaced |
| `ReplaceWith$` | SVar pointing to the replacement ability |
| `Prevent$ True` | Nothing happens instead (alternative to ReplaceWith) |
| `ReplacementResult$ Updated` | Modifies the event rather than replacing entirely |
| `Layer$` | Ordering: `CantHappen`, `Control`, `Copy`, `Transform`, `Other` |
| `ActiveZones$` | Zone the source must be in |
| `Skip$ True` | For phase/turn skipping |

### Replacement Types

| Event | When | Key Parameters |
|-------|------|----------------|
| `Moved` | Card would change zones | `ValidCard$`, `Origin$`, `Destination$` |
| `DamageDone` | Damage would be dealt | `ValidSource$`, `ValidTarget$`, `IsCombat$` |
| `Draw` | Player would draw | `ValidPlayer$` |
| `GainLife` | Player would gain life | `ValidPlayer$` |
| `AddCounter` | Counter would be added | `ValidCard$`, `CounterType$` |
| `CreateToken` | Token would be created | |
| `Destroy` | Permanent would be destroyed | `ValidCard$` |
| `BeginPhase` / `BeginTurn` | Phase/turn would begin | `Phase$`, `ValidPlayer$` |
| `GameLoss` | Player would lose | `ValidPlayer$` |
| `ProduceMana` / `LoseMana` | Mana events | |
| `Untap` | Permanent would untap | `ValidCard$` |

### Common Replacement Patterns

**ETB tapped:**
```
R:Event$ Moved | ValidCard$ Card.Self | Destination$ Battlefield | ReplacementResult$ Updated | ReplaceWith$ ETBTapped | Description$ CARDNAME enters tapped.
SVar:ETBTapped:DB$ Tap | Defined$ Self | ETB$ True
```

**Skip draw step:**
```
R:Event$ BeginPhase | ActiveZones$ Battlefield | ValidPlayer$ You | Phase$ Draw | Skip$ True | Description$ Skip your draw step.
```

**Prevent all damage to you:**
```
R:Event$ DamageDone | ActiveZones$ Battlefield | Prevent$ True | ValidTarget$ You | Description$ Prevent all damage that would be dealt to you.
```

### Accessed via Replaced Variables

```
SVar:X:ReplacedDamageAmount     -- Amount of damage being replaced
Defined$ ReplacedCard            -- Card being moved in a Moved replacement
```

## SVars

SVars (String Variables) are declared with the `SVar:` prefix and serve multiple purposes:

```
SVar:<Name>:<Value>
```

### SVars as Sub-Abilities

SVars frequently hold ability definitions referenced by `Execute$`, `SubAbility$`, `ReplaceWith$`, etc.:

```
SVar:TrigDmg:DB$ DealDamage | Defined$ TriggeredSourceController | NumDmg$ 2
SVar:DBPump:DB$ Pump | Defined$ Targeted | NumAtt$ 1 | NumDef$ 1
```

### SVars as Computed Values (Count$)

The `Count$` expression system evaluates dynamic numeric values:

```
SVar:X:Count$xPaid                          -- X mana paid
SVar:X:Count$Valid Creature.YouCtrl         -- Number of creatures you control
SVar:X:Count$YourLife                       -- Your life total
SVar:X:Count$Domain                         -- Number of basic land types you control
SVar:X:Count$CardsInYourHand               -- Cards in your hand
SVar:X:Count$CardsInYourGrave              -- Cards in your graveyard
SVar:X:Count$YourPoisonCounters            -- Your poison counters
SVar:X:Count$ValidGraveyard,Exile Instant.YouOwn,Sorcery.YouOwn  -- Instants/sorceries you own in graveyard+exile
```

Count expressions use the `Valid` syntax for filtering. The full set of counting functions is in `AbilityUtils.xCount()`.

### SVars for Card Properties

Access properties of specific objects:

```
SVar:X:TriggeredCard$CardPower              -- Power of triggered card
SVar:X:Remembered$Amount                    -- Number of remembered objects
SVar:X:TriggerCount$TriggeredLifeAmount     -- Life amount from trigger
```

### Referencing SVars in Parameters

SVars can be referenced wherever numeric values are expected. For example, `NumDmg$ X` will look up the SVar named `X` on the card.

## SubAbility Chaining

Abilities chain together using `SubAbility$` and `Execute$`:

### SubAbility$ (inline chaining)

```
A:SP$ Untap | ValidTgts$ Creature | SubAbility$ DBPump | SpellDescription$ Untap target creature. It gets +1/+1 until end of turn.
SVar:DBPump:DB$ Pump | Defined$ Targeted | NumAtt$ 1 | NumDef$ 1
```

The sub-ability resolves as part of the same resolution. `DB$` prefix means it cannot be played independently.

### Execute$ (trigger/replacement chaining)

Triggers and replacements use `Execute$` to point to their effect:

```
T:Mode$ Phase | Phase$ Upkeep | ValidPlayer$ You | Execute$ TrigDraw | TriggerDescription$ ...
SVar:TrigDraw:DB$ Draw | NumCards$ 2
```

### Multi-Level Chaining

Sub-abilities can chain further:

```
SVar:TrigPeek:DB$ PeekAndReveal | RememberRevealed$ True | SubAbility$ DBTransform
SVar:DBTransform:DB$ SetState | Mode$ Transform | ConditionDefined$ Remembered | ConditionPresent$ Card.Instant,Card.Sorcery | SubAbility$ DBCleanup
SVar:DBCleanup:DB$ Cleanup | ClearRemembered$ True
```

### Additional Named Sub-Abilities

Some effects use specific sub-ability parameters:

```
TrueSubAbility$ / FalseSubAbility$    -- Branch effect
WinSubAbility$ / OtherwiseSubAbility$ -- Clash, FlipACoin
RepeatSubAbility$                      -- Repeat/RepeatEach loops
```

## Restrictions and Conditions

Restrictions are checked when the ability would go on the stack. Conditions are checked on resolution.

### Restrictions

| Parameter | Purpose | Example |
|-----------|---------|---------|
| `SorcerySpeed$ True` | Only when you could cast a sorcery | `SorcerySpeed$ True` |
| `PlayerTurn$ True` | Only during your turn | `PlayerTurn$ True` |
| `OpponentTurn$ True` | Only during opponent's turn | |
| `ActivationLimit$ N` | N times per turn | `ActivationLimit$ 1` |
| `GameActivationLimit$ N` | N times per game | |
| `ActivationPhases$` | Only during certain phases | `ActivationPhases$ Main1,Main2` |
| `ActivationZone$` | Activate from a non-battlefield zone | `ActivationZone$ Graveyard` |
| `Planeswalker$ True` | Planeswalker timing restriction | |
| `Ultimate$ True` | Marks planeswalker ultimates (for AI) | |
| `IsPresent$` | Cards matching Valid must exist | `IsPresent$ Creature.YouCtrl` |
| `CheckSVar$` | SVar value check | `CheckSVar$ X` |
| `SVarCompare$` | Comparison for CheckSVar | `SVarCompare$ GE3` |

| Parameter | Purpose | Example |

## AI Hints

### Card-Level AI Removal (`AI:` line)

Controls whether the AI includes this card in generated decks:

```
AI:RemoveDeck:All               -- Never use in AI decks (card is unusable or harmful for AI)
AI:RemoveDeck:Random            -- Don't use in random decks (too narrow/situational)
AI:RemoveDeck:NonCommander      -- Don't use in non-Commander AI decks
```

### Ability-Level AI Parameters

Parameters added within ability definitions:

| Parameter | Purpose | Example |
|-----------|---------|---------|
| `AILogic$` | General AI behavior hint | `AILogic$ Burn` |
| `IsCurse$ True` | Marks positive effects used negatively | `IsCurse$ True` |
| `AITgts$` | Preferred AI targeting | `AITgts$ BetterThanEvalRating.130` |
| `AITgtsStrict$ True` | Only play if AITgts are available | |
| `AICheckSVar$` | AI checks SVar before playing | `AICheckSVar$ X` |

### Common AILogic$ Values

**Generic values** (handled in SpellAbilityAi base class):
- `Never` — AI will never play this ability
- `Once` — AI activates at most once per turn
- `Always` — AI always plays (even for normally unwilling triggers)

**Effect-specific values** (handled by individual XxxAi classes):
- `Burn`, `Triskelion` — damage targeting preferences
- `Pump`, `Curse` — buff/debuff AI behavior
- `InstantOrSorcery` — peek/reveal logic for spell types
- `DiscardLands`, `DigForCreature` — card selection preferences
- Many card-specific values (e.g., `Aristocrat`, `Polymorph`, `Pongify`)

### AI-Specific SVars

These SVars influence AI behavior without affecting game rules:

| SVar | Purpose | Example |
|------|---------|---------|
| `PlayMain1` | Play in first main phase | `SVar:PlayMain1:TRUE` |
| `NeedsToPlayVar` | Don't play unless condition met | `SVar:NeedsToPlayVar:Y GE3` |
| `SacMe` | Priority for being sacrificed (1-6, higher = more willing) | `SVar:SacMe:5` |
| `BuffedBy` | AI plays matching cards in Main1 | `SVar:BuffedBy:Enchantment.Aura` |
| `AntiBuffedBy` | AI plays matching cards vs opponent | `SVar:AntiBuffedBy:Enchantment.Aura` |
| `EnchantMe` | Priority for receiving auras (`Multiple`/`Once`) | `SVar:EnchantMe:Multiple` |
| `EquipMe` | Priority for receiving equipment (`Multiple`/`Once`) | `SVar:EquipMe:Once` |
| `MustAttack` | AI always attacks with this | `SVar:MustAttack:True` |
| `MustBeBlocked` | AI expects this to be blocked | `SVar:MustBeBlocked:True` |
| `HasCombatEffect` | Has a relevant combat ability | `SVar:HasCombatEffect:True` |
| `HasAttackEffect` / `HasBlockEffect` | Has attack/block triggered abilities | `SVar:HasAttackEffect:True` |
| `EndOfTurnLeavePlay` | AI knows this leaves at end of turn | `SVar:EndOfTurnLeavePlay:True` |
| `NonStackingEffect` | Second copy is useless | `SVar:NonStackingEffect:True` |
| `NoZeroToughnessAI` | AI avoids setting toughness to 0 | `SVar:NoZeroToughnessAI:True` |
| `UntapMe` | AI prioritizes untapping this | `SVar:UntapMe:True` |
| `AIEvaluationModifier` | Adjusts card evaluation score | `SVar:AIEvaluationModifier:50` |
| `AIPreference` | Cost payment preferences | `SVar:AIPreference:SacCost$Creature.token` |
| `ManaNeededToAvoidNegativeEffect` | Hold open mana for protection | |
| `Targeting` | Target selection hint | `SVar:Targeting:Dies` |

**NeedsToPlayVar example (Eldrazi Monument):**
```

```
AI won't play the card unless it controls 3+ creatures.

## Deck Building Hints

These help AI deck generators build synergistic decks.

### DeckHints

Increases ranking of cards sharing these traits during drafting/deck building:

```
DeckHints:Type$Instant|Sorcery           -- Wants instants/sorceries
DeckHints:Keyword$Flying                 -- Wants flyers
DeckHints:Ability$Proliferate            -- Wants proliferate cards
DeckHints:Color$Black                    -- Wants black cards
DeckHints:Name$Specific Card Name        -- Wants a specific card
```

### DeckNeeds

Stronger version — card is ranked lower unless deck contains cards matching these:

```
DeckNeeds:Type$Aura                      -- Needs auras in deck
DeckNeeds:Type$Human & Type$Warrior      -- Needs Human Warriors (both required)
DeckNeeds:Type$Human|Warrior             -- Needs Humans OR Warriors
DeckNeeds:Ability$Token|Counters         -- Needs token or counter synergy
```

### DeckHas

Declares what archetypes/abilities this card provides to the deck:

```
DeckHas:Ability$Token                    -- Provides token synergy
DeckHas:Ability$Counters                 -- Provides counter synergy
DeckHas:Ability$Graveyard               -- Provides graveyard synergy
DeckHas:Ability$Token & Ability$Counters -- Provides both
```

Combine multiple types with `&`. Use `|` for alternatives within a type.

## Conventions

### File Naming
- All lowercase, underscores for spaces, skip special characters
- Multi-face cards join names: `delver_of_secrets_insectile_aberration.txt`

### File Format
- Unix (LF) line endings
- No empty lines except before `ALTERNATE` sections
- Order: abilities → SVars → AI SVars → Oracle (last)
- Omit default parameter values for conciseness (e.g., just `SP$ Draw` instead of `SP$ Draw | Defined$ You | NumCards$ 1`)

### Text Substitution
- `CARDNAME` is replaced with the card's name at runtime
- `NICKNAME` is replaced with a shorter form in some contexts
- `\n` in Oracle text creates line breaks

### General
- Counter types are ALL CAPS: `P1P1`, `M1M1`, `CHARGE`, `LOYALTY`, `LORE`
- Keywords in `AddKeyword$` are `&`-delimited: `Flying & Haste`
- `CARDNAME` in costs refers to the card itself: `Sac<1/CARDNAME>`
