# Implementation Plan: Transformer Model Architecture

**Branch**: `007-transformer-model-arch` | **Date**: 2026-03-14 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-transformer-model-arch/spec.md`

## Summary

Add an encoder-only transformer model (~2M parameters) for MTG card price prediction that coexists with the existing sklearn model. The model tokenizes converted card script text (feature 006) using BERT WordPiece, processes it through 4 transformer encoder layers with attention, and predicts shifted-log prices. Training must complete on a GeForce RTX 3060 Ti (8 GB VRAM). Both models always run at the API evaluation endpoint; the transformer prediction gracefully degrades to `null` if the model artifact is unavailable.

## Technical Context

**Language/Version**: Python 3.14+
**Primary Dependencies**: PyTorch (`torch`), Hugging Face `transformers` (BERT tokenizer), existing: scikit-learn, FastAPI, uvicorn, numpy, pandas, joblib, ijson
**Storage**: `.pt` model artifact in `models/transformer/`, existing `.joblib` models in `models/`
**Testing**: pytest (unit + integration), ruff (linting)
**Target Platform**: Windows 11 (development), NVIDIA GPU with CUDA (training), CPU fallback (inference)
**Project Type**: CLI + web service (existing)
**Performance Goals**: Single-card inference < 100 ms, training on ~20K cards < 10 minutes on RTX 3060 Ti, median percentage error ≤ 50%
**Constraints**: Peak VRAM < 8 GB during training, CPU fallback for inference, deterministic predictions
**Scale/Scope**: ~20,000 training cards, single-model artifact, 2 new CLI subcommands, 1 modified API endpoint
