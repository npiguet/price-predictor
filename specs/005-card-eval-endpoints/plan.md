# Implementation Plan: Card Evaluation Endpoints

**Branch**: `005-card-eval-endpoints` | **Date**: 2026-03-01 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-card-eval-endpoints/spec.md`

## Summary

Add structured request logging to the existing `/api/v1/evaluate` endpoint (FR-012) and create a thin CLI subcommand (`eval`) that reads a Forge card script file and delegates evaluation to the running endpoint via stdlib HTTP. No new dependencies required — all changes build on existing infrastructure.

## Technical Context

**Language/Version**: Python 3.14+
**Primary Dependencies**: FastAPI, uvicorn (existing) — no new dependencies added
**Storage**: N/A (no storage changes)
**Testing**: pytest + FastAPI TestClient (existing)
**Target Platform**: Local machine (localhost/internal use)
**Project Type**: CLI + web-service (both)
**Performance Goals**: <3s endpoint response (SC-001), <5s CLI cycle (SC-002)
**Constraints**: No authentication, no new dependencies, UTF-8 only
**Scale/Scope**: Single-card evaluation per request
